# Fixed Controller Mappings

This feature uses position-first mapping. The goal is to make a modern pad feel correct by thumb position, not by printed letters.

Android standard logical face buttons:
- `BUTTON_A` = south
- `BUTTON_B` = east
- `BUTTON_X` = west
- `BUTTON_Y` = north

## SNES

Gameplay mapping:
- south -> `B`
- east -> `A`
- west -> `Y`
- north -> `X`
- `L1` -> `L`
- `R1` -> `R`
- `Start` -> `Start`
- `Select` or controller `Back` -> `Select`
- D-pad -> directions
- left stick -> directions

Result on an Xbox controller:
- `A` button acts as SNES `B`
- `B` button acts as SNES `A`
- `X` button acts as SNES `Y`
- `Y` button acts as SNES `X`

## Genesis

Gameplay mapping:
- west -> `A`
- south -> `B`
- east -> `C`
- `L1` -> `X`
- north -> `Y`
- `R1` -> `Z`
- `Start` -> `Start`
- `Select` or controller `Back` -> `Mode`
- D-pad -> directions
- left stick -> directions

Result on an Xbox controller:
- `X` button acts as Genesis `A`
- `A` button acts as Genesis `B`
- `B` button acts as Genesis `C`
- `LB` acts as Genesis `X`
- `Y` acts as Genesis `Y`
- `RB` acts as Genesis `Z`

## Explicitly Ignored In V1

- `L2`
- `R2`
- left stick click
- right stick click
- right stick axes
- guide/system/home button
- secondary controllers
